4 Régressions linéaires

Les régressions linéaires sont des régressions avec une variable dépendante métrique. Cela signifie que ce que vous voulez expliquer a des valeurs numériques.

Par exemple:

  • Comment le genre, l’âge et le statut socio-professionnel des individus influencent leur revenu ?

Dans ce cas, le revenu est ce que l’on veut expliquer. Si notre variable sur le revenu est métrique (par exemple, le revenu mensuel en $), alors on peut utiliser une régression linéaire.

Il existe plusieurs conditions à l’utilisation des régressions (cf. cours). Dans cette session, nous allons nous intéresser à l’application des régressions dans R.

4.1 Régressions linéraires simples

Tout d’abord, nous allons prendre l’exemple de régression le plus basique: la régression linéaire simple.

C’est une régression linéaire avec une seule variable indépendante. Elle répond à la question:

  • Quel est l’effet de X sur Y?
    • Y est donc la variable dépendante
    • X est donc la variable indépendante

Pour modéliser cela dans R, il faut utiliser la fonction lm(Y~X, données).

Par exemple, si on se demande, quel est l’effet de la proximité avec la frontière russe sur le soutien du parti pro-russe en 2014?

Dans ce cas, on peut utiliser les données UA_precints, dans lesquelles on prend:

  • pro_russian comme variable dépendante
  • within_25km comme variable indépendante

Dans R on fait:

Code 
model_1 <- lm(pro_russian~within_25km, UA_precincts)

Les résultats de notre régression sont maintenant assignés à l’objet model_1.

Nous avons maintenant fait notre régression linéaire simple ; il faut maintenant pouvoir interpréter les résultats.

4.2 Comprendre l’output d’une régression linéaire.

L’objet model_1 contient beaucoup d’informations. Heureusement, nous pouvons utiliser la fonction summary() pour extraire les informations principales de cet objet.

Code 
summary(model_1)

Coefficients:

(Intercept)  25.9765     0.3431  75.714  < 2e-16 ***
within_25km   4.8934     0.8820   5.548  3.1e-08 ***

Multiple R-squared:  0.008507,  Adjusted R-squared:  0.008231 
F-statistic: 30.78 on 1 and 3587 DF,  p-value: 3.103e-08

Dans l’output on a 4 types d’information:

  • La partie Call donne la formule que nous avons estimée.
  • La partie Residuals présente la distribution des résidus (erreurs de prédiction).
  • La partie Coefficients, nous indique les informations principales pour un tableau de régression.
  • Le bas du summary donne des informations générales sur le modèle (écart-type des résidus, coefficient de détermination R Squared, F-score et significativité du F-Score).

Toutes ces informations sont importantes (cf. cours), cependant, nous allons dans cette session nous attarder sur la partie Coefficients qui nous présente les informations importantes pour comprendre notre régression.

Dans cette partie, on a un tableau qui a 5 colonnes et des ***. Dans l’ordre ces informations indiquent:

  • Les variables incluses dans le modèle. Dans ce cas, nous avons simplement une constante (Intercept) et within_25km, qui est notre variable indépendante.

  • Estimate présente les coefficients de régression. Ces coefficients indiquent l’effet de nos variables indépendantes sur notre variable dépendante. Dans notre cas on a:

    • (Intercept) = 25.98. Cela indique que quand notre variable indépendante = 0, la valeur prédite de notre variable dépendante est 25.98.
    • within_25km = 4.98. Cela nous indique l’effet de notre variable indépendante sur notre variable dépendante. Elle indique de combien la valeur prédite de notre variable dépendante augmente quand notre variable dépendante augmente de 1.

  • Std. Error indique l’écart-type de notre coefficient. C’est sur cette base que sont calculées la t-value et la p-value.

  • t value indique le score du t-test.

  • Pr(>|t|) indique la p-value du coefficient. Cette valeur teste les hypothèses suivantes:

    • H0: la variable n’a pas d’effet.
    • H1: la variable a un effet.

  • Éventuellement: Les étoiles. Indique le test de significativité que le coefficient passe, basé sur la légende en bas du tableau.

    • Signif. codes: 0 ‘***’ 0.001 ‘**’ 0.01 ‘*’ 0.05 ‘.’ 0.1 ‘ ’ 1
    • Ce qui signifie:
      • Entre 0 et 0.001 ***, significatif au-dessus de 99.9%
      • Entre 0.001 et 0.01 **, significatif au-dessus de 99% mais pas de 99.9%
      • Entre 0.01 et 0.05 *, significatif au-dessus de 95% mais pas de 99%
      • Entre 0.05 et 0.1 +, significatif au-dessus de 90% mais pas de 95%

Dans notre cas, on voit donc que l’effet de la variable within_25km est positif (coefficient de régression au-dessus de 0) et significatif à plus de 99,9 % (p-value < 2.2e10^-16, et ***).

4.3 Régression linéaire multiple

Il est rare que l’on s’arrête à des régressions simples - avec une seule variable indépendante. En pratique, les régressions permettent de tester l’effet conjoint de plusieurs variables, en tenant d’autres variables sous contrôle.

Par exemple, nous avons vu dans la dernière session que la proximité des districts avec la frontière russe n’est pas indépendante de la réception de la télévision russe. Le tableau et le test du chi carré présentés ci-dessous nous résument cela.

Code 
table(UA_precincts$russian_tv, UA_precincts$within_25km)
   
       0    1
  0 2725  167
  1  321  376
Code 
chisq.test(UA_precincts$russian_tv, UA_precincts$within_25km)

    Pearson's Chi-squared test with Yates' continuity correction

data:  UA_precincts$russian_tv and UA_precincts$within_25km
X-squared = 1011.2, df = 1, p-value < 2.2e-16

On a donc une distribution inégale de la télévision russe dans les districts proches de la frontière et dans ceux plus éloignés de la frontière. Pour aller plus loin, on peut aussi comparer la proportion des districts proches et éloignés de la frontière qui reçoivent la télévision russe:

Code 
# proportion des districts à plus de 25 km de la frontière
prop.table(table(UA_precincts[UA_precincts$within_25km==0,]$russian_tv)) 

        0         1 
0.8946159 0.1053841
Code 
# proportion des districts à moins de 25 km de la frontière
prop.table(table(UA_precincts[UA_precincts$within_25km==1,]$russian_tv))

        0         1 
0.3075506 0.6924494

On voit que seuls 10% des districts à plus de 25 km de la frontière reçoivent la télévision russe, alors que près de 70% des districts proches de la frontière la reçoivent. Dans ce cas, il est important de se demander si c’est la proximité avec la frontière russe ou la réception de la télévision russe qui influence le vote pour le parti pro-russe.

Pour ce faire on peut faire une régression linéaire multivariée avec:

  • pro_russian comme variable dépendante
  • within_25km comme première variable indépendante
  • russian_tv comme deuxième variable indépendante

En pratique, dans R, on additionne les effets des variables. On a donc:

Code 
model_2 <- lm(pro_russian~within_25km+russian_tv, UA_precincts)
summary(model_2)

Call:

Residuals:

-41.117 -15.707  -4.768  14.137  54.877 

Coefficients:

(Intercept)  24.0247     0.3375  71.181  < 2e-16 ***

Les résultats du model_2 présentent une image très différente du model_1 en effet on observe que:

  • Sous contrôle de la réception de la télévision russe, la proximité avec la frontière russe diminue le vote pour le parti pro-russe.
  • La réception de la télévision russe augmente le vote pour le parti pro-russe.

Les régressions linéaires multivariées sont donc des outils puissants pour estimer l’effet de certaines variables sur d’autres variables.

Cependant, en les utilisant, on modélise une version de la réalité qui n’est pas conforme avec la réalité qui est bien plus complexe.

Le but d’une régression est de s’assurer que l’effet de X sur Y que l’on observe est bien dû à X et non à une variable omise Z.

Une phase souvent citée peut se paraphraser par: Toutes les régressions sont fausses, mais certaines sont utiles.

In fine, en sciences sociales, les régressions présentent une modélisation simplifiée des réalités sociales. Cependant, ce n’est pas parce que cette modélisation est simplifiée qu’elle n’est pas utile pour comprendre certains phénomènes sociaux.

Avant de faire vos régressions, demandez-vous: au-delà des variables incluses dans mes modèles, y a-t-il un autre élément qui puisse influencer la variance de ma variable dépendante ?

5 Les interactions

Jusqu’à présent, nous avons testé l’effet direct de nos variables indépendantes sur notre variable dépendante. Cependant, on peut aussi considérer qu’il y a certains phénomènes qui sont influencés par l’interaction entre plusieurs variables.

Par exemple:

  • L’effet du genre et de la classe d’âge sur le revenu.
  • La classe d’âge n’a pas le même effet sur le revenu pour les hommes ou pour les femmes (par exemple maternité).
  • On peut donc considérer et émettre l’hypothèse d’un effet d’interaction entre la classe d’âge et le genre sur le revenu.

Dans notre cas, on peut aussi se demander si l’effet de réceptionner la télévision russe sur le vote pour le parti pro-russe est le même pour les districts proches et éloignés de la frontière.

Pour répondre à cette question, il faut inclure un terme d’interaction dans notre modèle.

En pratique, cela revient à multiplier des variables entre elles.

Code 
model_3 <- lm(pro_russian~within_25km*russian_tv, UA_precincts)
summary(model_3)

Call:
lm(formula = pro_russian ~ within_25km * russian_tv, data = UA_precincts)

Residuals:
    Min      1Q  Median      3Q     Max 
-44.235 -15.401  -5.101  14.048  55.244 

Coefficients:
                       Estimate Std. Error t value Pr(>|t|)    
(Intercept)             23.6574     0.3410  69.383  < 2e-16 ***
within_25km              0.3813     1.4189   0.269    0.788    
russian_tv              22.0063     1.0503  20.952  < 2e-16 ***
within_25km:russian_tv -12.1409     1.9603  -6.193 6.56e-10 ***
---
Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1

Residual standard error: 17.8 on 3585 degrees of freedom
Multiple R-squared:  0.1244,    Adjusted R-squared:  0.1237 
F-statistic: 169.8 on 3 and 3585 DF,  p-value: < 2.2e-16

Quand on inclut un terme d’interaction dans notre modèle entre une variable 1 et une variable 2 on va avoir:

  • L’effet direct de la variable 1.
  • L’effet direct de la variable 2.
  • L’effet d’interaction de la variable 1 et de la variable 2.

Dans notre cas on a:

  • L’effet direct de la variable within_25km.
  • L’effet direct de la variable russian_tv.
  • L’effet d’interaction de la variable within_25km et de la variable russian_tv.

Cependant, cet effet est compliqué à interpréter directement car il faut considérer la multiplication des deux variables à la fois, et tenir compte aussi des effets directs.

Dans notre cas, les deux variables sont binaires (0 ou 1) ; il est donc facile de comprendre que quand les deux variables sont égales à 1, la valeur prédite de la variable dépendante diminue de 12.14. Dans tous les autres cas, la multiplication des variables est égale à 0.

Cependant, pour faciliter l’interprétation, on peut aussi calculer des valeurs prédites de la variable dépendante sous différents scénarios. Pour ce faire, on peut notamment utiliser la fonction predict().

Le plus simple pour ce faire est de:

  1. Créer des fausses données avec les valeurs des variables dépendantes que l’on veut prédire.
  2. Utiliser la fonction predict() pour prédire les valeurs de la variable dépendante
Code 
# 1. Créer des données fictives avec les cas que l'on veut prédire
new_data <- as.data.frame(list(within_25km = c(0,0,1,1),
                               russian_tv=c(0,1,0,1),
                               fit = NA, 
                               lower = NA, 
                               upper = NA)) # Cela nous sert pour stocker les prédictions par la suite

# 2. Prédire les valeur de la variable dépendantes selon l'objet model_3
new_data[,3:5] <- predict(model_3, newdata = new_data, interval = "confidence")

new_data
  within_25km russian_tv      fit    lower    upper
1           0          0 23.65737 22.98886 24.32589
2           0          1 45.66363 43.71584 47.61142
3           1          0 24.03864 21.33819 26.73909
4           1          1 33.90397 32.10427 35.70367

Voilà, à présent nous avons les valeurs prédites pour les différents cas qui nous intéressent. On peut bien sûr analyser les différents chiffres. Dans ce cas on voit que:

  • Le soutien prédit pour le parti pro-russe le plus haut est dans les districts qui reçoivent la télévision russe (russian_tv=1) et qui sont éloignés de la frontière russe (within_25_km=0).
  • Le soutien prédit pour le parti pro-russe le plus bas est dans les districts qui ne reçoivent pas la télévision russe (russian_tv=1) et qui sont éloignés de la frontière russe (within_25_km=0).

Cependant, il est parfois plus pratique de communiquer ces résultats dans un graphique:

Code 
library(ggplot2)

ggplot(data = new_data)+
  geom_point(aes(x=factor(within_25km), y = fit, color = factor(russian_tv)))+
  geom_errorbar(aes(x=factor(within_25km), ymin = lower, ymax=upper, color = factor(russian_tv)))

Et puis, on peut le styliser un peu:

Code 
ggplot(data = new_data)+
  geom_point(aes(x=factor(within_25km), y = fit, color = factor(russian_tv)))+
  geom_errorbar(aes(x=factor(within_25km), ymin = lower, ymax=upper, color = factor(russian_tv)), width = .1)+
  scale_x_discrete(name = "Distance de la frontière russe", labels = c("Plus de 25 km", "Moins de 25 km"))+
  scale_color_manual(name = "",labels = c("Ne recoit pas la TV russe", "Recoit la TV russe"),
                     values = c("black", "grey"))+
  ylab("Valeur prédite de soutien au \n parti pro-russe en 2014 \n (en %)")+
  theme_minimal()+
  theme(legend.position = "bottom")

Grâce à ces analyses, on voit que l’effet de la proximité avec la frontière russe est conditionné à la réception de la télévision russe.

Pour les districts qui ne reçoivent pas la télévision russe, il n’y a pas de différence. Pour les districts qui reçoivent la télévision russe, on voit que le soutien est plus élevé dans les districts qui sont éloignés de la frontière.

6 Les variables catégorielles dans les régressions

À présent, nous allons examiner le cas particulier des variables catégorielles.

Imaginons que nous souhaitons estimer l’effet du taux de personnes ayant une haute éducation sur le soutien au Brexit dans les districts. Cependant, nous souhaitons traiter le taux de personnes avec une haute éducation comme une variable catégorielle dans le modèle, en fonction du quartile dans lequel le district s’inscrit.

Premièrement, regardons la distribution de cette variable:

Code 
summary(UK_districts$high_education)
   Min. 1st Qu.  Median    Mean 3rd Qu.    Max.    NA's 
  14.21   21.77   25.76   26.92   30.76   68.36       2

En regardant cette distribution, on peut créer notre variable en quatre catégories:

Code 
library(dplyr)

Attaching package: 'dplyr'
The following objects are masked from 'package:stats':

    filter, lag
The following objects are masked from 'package:base':

    intersect, setdiff, setequal, union
Code 
UK_districts <- UK_districts %>% 
  mutate(age_group = case_when(high_education<21.77 ~ "1st quartile", 
                               high_education>=21.77 & high_education<25.76 ~ "2nd quartile", 
                               high_education>=25.76 & high_education<30.76 ~ "3rd quartile", 
                               high_education>=0.76 ~ "4th quartile"))

À présent, nous allons faire un modèle de régression linéaire avec la variable leave comme variable dépendante et la variable age_group comme variable indépendante.

model_4 <- lm(leave~age_group, UK_districts)

Alors que l’on a mis une seule variable dans notre formule lm(leave~age_group, UK_districts), on a trois variables dans le tableau de régression.

R comprend que notre variable est une variable catégorielle, avec quatre catégories. Il inclut donc l’équivalent de trois variables binaires qui indiquent la distribution des quatre catégories de la manière suivante:

  • Si age_groupe2nd quartile est 1 alors age_group3rd quartile et age_group4th quartile sont 0.
  • Si age_groupe3rd quartile est 1 alors age_group2nd quartile et age_group4th quartile sont 0.
  • Si age_groupe4th quartile est 1 alors age_group2nd quartile et age_group3rd quartile sont 0.
  • Si age_groupe2nd quartile, age_group3rd quartile et age_group4th quartile sont 0, alors le groupe est obligatoirement dans le premier quartile de la distribution.

Pour interpréter les coefficients de régression, il faut toujours se référer à la catégorie de référence (celle qui n’apparaît pas dans le modèle).

Dans notre cas on voit que:

  • Les districts du 2e quartile ont 5,3 % de voix pour le Brexit en moins par rapport aux districts du premier quartile.
  • Les districts du 3e quartile ont 10,1 % de voix pour le Brexit en moins par rapport aux districts du premier quartile.
  • Les districts du 4e quartile ont 19,3 % de voix pour le Brexit en moins par rapport aux districts du premier quartile.

7 Les régressions logistiques

Il existe d’autres types de régressions que les régressions linéaires. Le type de régression que l’on choisit dépend du type de variable dépendante que nous avons, ainsi que de sa distribution.

Un cas fréquent qui ne correspond pas au cadre du modèle linéaire est la variable binaire.

Par exemple:

  • Voter ou non pour ce parti
  • Aller au travail en voiture ou non
  • Faire des courses en ligne ou non

Il existe des tas de phénomènes sociaux binaires que l’on peut souhaiter analyser. Dans ce cas, on va faire une régression logistique. Cela revient à opérer une transformation logit de l’équation de regression selon la formule:

\[logit(x) = \frac{e^x}{1+e^x}\]

Cette transformation change un nombre continu entre -infini et +infini en un nombre entre 0 et 1. Cela est utilisé dans les régressions logistiques pour indiquer la probabilité que la variable dépendante soit égale à 1.

Si l’on compare la distribution logit d’une variable à sa prédiction selon un modèle linéaire on a:

Code 
ggplot()+
  geom_point(aes(x=c(-500:500)/100, y = exp(c(-500:500)/100)/(1+exp(c(-500:500)/100))))+

Dans ce cas, les points noirs indiquent la distribution logit de la variable. Si on essaie d’estimer ces valeurs avec un modèle linéaire on voit plusieurs problèmes, notamment que:

  • Des valeurs supérieures à 1 sont possibles.
  • Des valeurs inférieures à 0 sont possibles.

En fait, la régression logistique indique l’effet d’un groupe de variables sur la probabilité que la variable dépendante = 1.

Dans R, pour faire une régression logistique il faut utiliser la fonction glm(y~x, données, family=binomial(link="logit")).

La première partie de la fonction (glm(y~x, données)) se note exactement comme pour une régression linéaire.

La seconde partie indique la transformation (family) avec laquelle on souhaite faire notre régression. Dans notre cas, c’est une transformation binomiale logit, qui s’indique family=binomial(link="logit").

Prenons l’exemple du soutien au Brexit dans les données BES. Le soutien au Brexit est codé 1, si l’individu vote pour le Brexit et 0 s’il s’y oppose.

La regression logistique nous permet d’estimer comment l’âge et le niveau d’éducation affectent la probabilité de voter pour le Brexit:
