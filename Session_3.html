2 Objectifs de la session

À la fin de cette session, vous serez capables de :

  • analyser la relation entre deux variables,
  • choisir un test statistique adapté,
  • utiliser R pour réaliser ces analyses,
  • interpréter correctement les résultats.

Les analyses bivariées permettent de passer d’une simple description des données à une réflexion sur les relations entre phénomènes sociaux.

3 Introduction: analyser des relations

Dans les sessions précédentes, nous avons étudié les variables une par une. Il s’agissait d’analyses univariées.

Dans cette session, nous cherchons à comprendre comment deux variables sont liées.

La question centrale devient :

  • Existe-t-il un lien entre X et Y ?

Exemples :

  • Le genre influence-t-il le vote ?
  • Le niveau d’études est-il lié au revenu ?
  • Le temps de travail est-il associé aux résultats scolaires ?

De plus, certains tests statistiques sur deux variables permettent de savoir:

  • Est-ce que le lien entre deux variables est statistiquement significatif () ?
  • Quel est le lien entre deux variables ?

4 Tableaux croisés: fréquences et proportions

Un tableau croisé permet d’étudier la relation entre deux variables catégorielles.

Il présente :

  • les effectifs par catégorie,
  • les proportions par ligne ou par colonne.

Les tableaux croisés constituent souvent la première étape de l’analyse bivariée.

4.1 Importer les données

Avant de donner des exemples concrets, nous pouvons importer les six bases de données présentées lors de la session 2.

Code 
STAR <- read.csv(url("https://raw.githubusercontent.com/ellaudet/DSS/refs/heads/master/STAR.csv"))

BES <- read.csv(url("https://raw.githubusercontent.com/ellaudet/DSS/refs/heads/master/BES.csv"))
UK_districts <- read.csv(url("https://raw.githubusercontent.com/ellaudet/DSS/refs/heads/master/UK_districts.csv"))

countries <- read.csv("https://raw.githubusercontent.com/ellaudet/DSS/refs/heads/master/countries.csv")

UA_survey <- read.csv(url("https://raw.githubusercontent.com/ellaudet/DSS/refs/heads/master/UA_survey.csv"))
UA_precincts <- read.csv(url("https://raw.githubusercontent.com/ellaudet/DSS/refs/heads/master/UA_precincts.csv"))

4.2 Tableaux croisé de fréquence

Pour vraiment naviguer dans ses données, il est important de savoir identifier les observations qui ont plusieurs caractéristiques simultanées. Par exemple:

  • Combien de personnes de plus de 60 ans ont voté pour le Brexit (données BES) ?
  • Combien de districts avec accès à la télévision russe se situent à plus de 25 km de la frontière russe (données UA_precincts)?

Pour ce faire, savoir naviguer dans les données et, notamment, comprendre les tableaux de fréquences est essentiel.

Code 
library(dplyr)

Attaching package: 'dplyr'
The following objects are masked from 'package:stats':

    filter, lag
The following objects are masked from 'package:base':

    intersect, setdiff, setequal, union
Code 
## Exemple 1: données BES
BES <- BES %>% 
  mutate(age60plus = ifelse(age>60, 1, 0))

table(BES$age60plus, BES$leave)
   
        0     1
  0 10347  7806
  1  4005  5886

Pour l’exemple 1, on voit qu’il y a 5886 personnes âgées de plus de 60 ans (age60plus==1) qui ont soutenu le Brexit (leave==1).

Code 
## Exemple 2: données UA_precincts

table(UA_precincts$russian_tv, UA_precincts$within_25km)
   
       0    1
  0 2725  167
  1  321  376

Dans ce cas, il est plus difficile de déterminer quel nombre est correct. En effet, on cherche les districts qui reçoivent la télévision russe (russian_tv=1) et qui ne se trouvent pas dans un périmètre de 25 km de la frontière russe (within_25km=0). On ne peut donc pas directement savoir si on doit regarder la deuxième colonne de la première ligne ou la deuxième ligne de la deuxième colonne.

Dans R, la première variable correspondra à la première dimension de l’objet, la deuxième à la deuxième, la troisième à la troisième, et ainsi de suite. Cependant, il est toujours possible de se tromper.

Autrement il est aussi possible de faire:

Code 
table(UA_precincts[UA_precincts$within_25km==0,]$russian_tv)

   0    1 
2725  321

Dans ce cas, on fait un tableau de fréquences d’une variable, mais on précise qu’on ne veut que les lignes pour lesquelles la valeur de la variable within_25km==0.

Avec cela, on peut rapidement constater qu’il y a 321 districts qui reçoivent la télévision russe en dehors du périmètre de 25 km autour de la frontière russe. Cependant, le nombre d’observations de notre tableau ne comprend que celles des districts situés hors du périmètre de 25 km autour de la frontière russe. Cela peut être important en fonction des proportions que l’on souhaite représenter.

Une troisième manière de procéder serait de créer une nouvelle variable qui croise les deux variables existantes:

Code 
UA_precincts <- UA_precincts %>% 
  mutate(russian_tv_and_border = case_when(russian_tv==1 & within_25km==1 ~ "Reçoit la TV et est dans le périmètre de 25km",
                                           russian_tv==0 & within_25km==1 ~ "Ne reçoit pas la TV et est dans le périmètre de 25km",
                                           russian_tv==1 & within_25km==0 ~ "Reçoit la TV et n'est pas dans le périmètre de 25km",
                                           russian_tv==0 & within_25km==0 ~ "Ne reçoit pas la TV et n'est pas dans le périmètre de 25km"))

table(UA_precincts$russian_tv_and_border)

      Ne reçoit pas la TV et est dans le périmètre de 25km 
                                                       167 
Ne reçoit pas la TV et n'est pas dans le périmètre de 25km 
                                                      2725 
             Reçoit la TV et est dans le périmètre de 25km 
                                                       376 
       Reçoit la TV et n'est pas dans le périmètre de 25km 
                                                       321

De cette manière, on voit clairement que 321 districts reçoivent la TV russe et ne sont pas dans le périmètre de 25 km.

4.3 Les proportion croisées

Pour les proportions, la même logique que pour les fréquences s’applique ; on peut simplement utiliser la fonction prop.table() et avoir les proportions associées. Cependant, il faut savoir quelle est la proportion qui nous intéresse. On peut différencier deux cas:

  • Quelle proportion des observations a x=x et y=y ?
  • Quelle proportion des observations avec x=x a y=y ?

Ces deux formulations changent catégoriquement notre calcul. Dans notre exemple on a:

  • Quelle proportion des districts reçoit la télévision russe et est en dehors du périmètre de 25 km de la frontière russe ?

Ou

  • Quelle proportion des districts qui reçoivent la télévision russe est en dehors du périmètre de la frontière russe ?

Ou

  • Quelle proportion des districts qui sont en dehors du périmètre de la frontière russe reçoit la télévision russe ?

Dans les 3 cas, la fréquence de ces districts est la même, mais les proportions sont très différentes:

Code 
# Premier cas
prop.table(table(UA_precincts$russian_tv_and_border))

      Ne reçoit pas la TV et est dans le périmètre de 25km 
                                                0.04653107 
Ne reçoit pas la TV et n'est pas dans le périmètre de 25km 
                                                0.75926442 
             Reçoit la TV et est dans le périmètre de 25km 
                                                0.10476456 
       Reçoit la TV et n'est pas dans le périmètre de 25km 
                                                0.08943996

On voit que sur tous les districts, seuls 8.9% sont en dehors du périmètre de 25 km autour de la frontière russe et reçoivent la télévision russe.

Code 
# Deuxième cas
prop.table(table(UA_precincts[UA_precincts$russian_tv==1,]$within_25km))

        0         1 
0.4605452 0.5394548

Cela nous indique que parmi les districts qui reçoivent la télévision russe, 46% sont en dehors du périmètre de 25 km de la frontière russe.

Code 
# Troisième cas
prop.table(table(UA_precincts[UA_precincts$within_25km==0,]$russian_tv))

        0         1 
0.8946159 0.1053841

En revanche, parmi les districts qui sont en dehors du périmètre de 25 km de la frontière russe, seuls 10% reçoivent la télévision russe.

5 Visualisation bivariée

5.1 Pourquoi utiliser des graphiques ?

Les graphiques permettent de :

  • visualiser les relations,
  • détecter des tendances,
  • repérer des valeurs atypiques,
  • compléter les tests statistiques.

Les nuages de points permettent de visualiser la relation entre deux variables.

Prenons l’exemple de la base de données UK_districts. On veut visualiser le lien entre le taux de personnes avec une formation supérieure et le taux de vote pour le Brexit dans les districts.

5.3 Une variable continue et une variable catégorielle : boxplots

Les boxplots permettent de visualiser rapidement la distribution d’une variable métrique selon différentes catégories.

Prenons le cas des données BES. Imaginons que l’on souhaite visualiser la distribution de l’âge des répondants selon leur niveau d’éducation. Dans ce cas on peut faire

Code 
ggplot(BES, aes(x = factor(education), y = age)) +
  geom_boxplot()

Ainsi, on peut voir la distribution de l’âge médian des différents niveaux d’éducation, l’espace interquartile et les outliers en un coup d’œil.

Rappel: comme la variable education est une variable métrique mais que l’on souhaite la traiter comme une variable catégorielle, on utilise la fonction factor().

5.4 Deux variables catégorielles: Bar plot multiples.

Pour visualiser simultanément les fréquences de deux variables catégorielles, on peut utiliser un diagramme en barres multiples. Pour ce faire, on peut par exemple préciser que les barres ont un remplissage différent selon notre deuxième variable.

Par exemple, si l’on veut savoir simultanément combien de personnes ont eu moins de 600 points, de 600 à 649 points ou 650 points ou plus pour les tests de maths et de lecture dans la base de données STAR, on peut faire:

Code 
# Etape 1, recoder les variables
STAR <- STAR %>% 
  mutate(math3 = case_when(math<600 ~ "Bas",
                           math>=600 & math<650 ~ "Moyen",
                           math>=650 ~ "Haut"),
         reading3 = case_when(reading<600 ~ "Bas",
                           reading>=600 & reading<650 ~ "Moyen",
                           reading>=650 ~ "Haut"))

# Etape 2, produire le graph: 
ggplot(STAR, aes(x=math3, fill = reading3))+
  geom_bar()

On peut aussi représenter ces catégories côte à côte:

Code 
ggplot(STAR, aes(x=math3, fill = reading3))+
  geom_bar(position = "dodge")

… et bien sûr ajouter des éléments estétiques:

Code 
ggplot(STAR, aes(x=factor(math3,
                          levels = c("Bas", "Moyen", "Haut")), 
                 fill = factor(reading3,
                               levels = c("Bas", "Moyen", "Haut"))))+
  geom_bar(position = "dodge", color = "black")+
  scale_fill_manual(name = "Score de lecture", values = c("white", "grey", "black"))+
  xlab("Score de math")+
  theme_minimal()

6 Test du Chi-deux (χ²)

6.1 Quand utiliser le test du χ² ?

Le test du chi carré est utilisé lorsque l’on étudie deux variables catégorielles.

Le but du test du chi carré est de tester la relation de dépendance entre deux variables catégorielles, à partir de la distribution des observations.

Concrètement, on teste les hypothèses suivantes

  • H0 : les variables sont indépendamment distribuées entre elles.
  • H1 : la distribution des observations des deux variables est dépendante.

Pour interpréter les résultats du test, on regarde la significativité du test, la valeur p. Cela indique la probabilité que l’hypothèse nulle soit vérifiée. On ne peut jamais être sûr à 100%, mais en dessous de certains seuils, on va considérer qu’on peut rejetter l’hypothèse nulle H0, donc que la distribution des observations n’est pas indépendante entre les deux variables.

Prenons l’exemple du diagramme de barres ci-dessus. On veut savoir si les résultats de maths et de lecture sont dépendants l’un de l’autre. Dans ce cas, on peut faire:

Code 
chisq.test(table(STAR$math3, STAR$reading3))

    Pearson's Chi-squared test

data:  table(STAR$math3, STAR$reading3)
X-squared = 578.03, df = 4, p-value < 2.2e-16

Pour ce test, on voit que la p-value est inférieure à 2.2e-16 (2.2 * 10^-16, on ajoute donc 16 0 avant 2.2). Cela représente le plus petit nombre que R peut calculer. La probabilité d’indépendance entre les deux variables est donc excessivement faible.

Il est fréquent que l’on parle de _seuil de significativité. Habituellement, un seuil de significativité est acceptable quand p<0.05 (en dessous de 5% de probabilité, donc significatif à plus de 95%). Cependant, d’autres seuils sont fréquemment utilisés:

  • p<0.01 (moins de 1%, significatif à plus de 99%)
  • p<0.001 (moins de 0.1% significatif à plus de 99.9%)
  • parfois: p<0.1 (moins de 10%, significatif à plus de 90%).

7 Corrélations

7.1 Principe

La corrélation mesure l’intensité et le sens d’une relation linéaire entre deux variables quantitatives. L’indice de corrélation est souvent cité dans les médias car c’est une mesure facilement communicable.

Cependant, cette mesure a d’importantes limitations.

  • Premièrement, une corrélation forte entre les variables n’équivaut pas à une causalité.
  • Deuxièmement, une corrélation forte dépend fortement du sujet d’étude.
    • Ex. : On n’attend pas la même corrélation entre des phénomènes sociaux et des phénomènes physiques.
  • Troisièmement, on peut observer une corrélation entre deux variables qui est en fait liée à une troisième variable non comprise dans l’analyse.

Il existe différents types de corrélation suivant le type de variable que vous considérez

  • Pearson: variables métriques
  • Spearman: variables ordinales

La corrélation de Pearson est la plus couramment utilisée. On peut par exemple mesurer la corrélation entre le niveau moyen d’émission de lumière pendant la nuit entre 1992 et 1993, et entre 2005 et 2006 dans la base de données countries en utilisant la fonction suivante:

Code 
cor(countries$light, countries$prior_light, use = "pairwise.complete.obs") ## use = "pairwise.complete.obs" indique que l'on veut inclure que les données qui sont complètes dans les deux variables. 
[1] 0.9730701

On voit que l’indice de corrélation entre ces deux variables est supérieur à 0.97. L’indice de corrélation s’interprète comme suit:

  • Plus la valeur est proche de 1, plus la corrélation est positive et forte.
  • Plus la valeur est proche de 0, plus on a une absence de relation.
  • plus la valeur est proche de -1 plus on a une corrélation négative forte.

De plus, on peut tester la significativité de cette corrélation avec:

Code 
cor.test(countries$light, countries$prior_light, use = "pairwise.complete.obs")

    Pearson's product-moment correlation

data:  countries$light and countries$prior_light
t = 54.715, df = 168, p-value < 2.2e-16
alternative hypothesis: true correlation is not equal to 0
95 percent confidence interval:
 0.9637002 0.9800459
sample estimates:
      cor 
0.9730701

À nouveau, on regarde la p-value, qui indique la probabilité que l’hypothèse nulle soit vraie. Dans ce cas, l’hypothèse nulle est l’absence de relation.

Dans notre cas on voit que: - La p-value est en dessous de 0.001. - Cela signifie que la corrélation positive est significative à plus de 99.9%.

8 Comparaison de moyennes

Le test t permet de comparer la moyenne d’une variable quantitative entre deux groupes. En pratique, le test t compare la moyenne d’une variable métrique et une variable catégorielle à deux catégories

Exemple : comparer les revenus selon le genre.

Dans ce cas on teste les hypothèses suivantes: - H0 : les moyennes sont égales - H1 : les moyennes sont différentes

La p-value indique donc la probabilité que les moyennes soient égales. Si on a un _p-value faible, on rejette H0 et on accepte H1, ce qui signifie que les moyennes sont différentes.

On peut par exemple se demander si les niveaux moyens d’émission de lumière pendant la nuit entre 1992 et 1993, et entre 2005 et 2006 ont une moyenne différente. Dans ce cas on peut faire :

Code 
# Étape 1: Reshape les données

# On a besoin d'une variable qui indique la variable métrique et une variable qui indique le groupe: 
countries_long <- as.data.frame(list(light = c(countries$prior_light, countries$light), ## Crée un vecteur avec les deux variables
                                     year = rep(c("1992-1993", "2005-2006"), each = nrow(countries)))) # crée un vecteur qui repète les valeurs du vecteur c("1992-1993, 2005-2006") d'une longueur nrow(countries) pour chaque observation

# Concrètement, on aura 1992-1993 répétés pour le nombre de lignes dans les données countries puis 2005-2006 répétés de manière similaire.

# Pour faire le t-test il faut faire la commande t.test(variable métrique ~ variable catégorielle, données) 

# Dans notre cas on a donc: 

t.test(light ~ year, data = countries_long)

    Welch Two Sample t-test

data:  light by year
t = -0.96592, df = 337.35, p-value = 0.3348
alternative hypothesis: true difference in means between group 1992-1993 and group 2005-2006 is not equal to 0
95 percent confidence interval:
 -1.8676977  0.6375036
sample estimates:
mean in group 1992-1993 mean in group 2005-2006 
               2.950330                3.565427

Pour interpréter les résultats on doit s’intéresser:

  • aux moyennes dans les groupes
  • à la p-value
  • Éventuellement à l’intervalle de confiance.

Dans notre cas on voit que la moyenne du niveau d’émission de lumière dans les pays est:

  • 2.95 entre 1992 et 1993
  • 3.57 entre 2005 et 2006

Dans ce cas, la différence de moyenne est de:

  • 3.57-2.95 = 0.62 ou…
  • 2.95-3.57 = -0.62

Ces deux valeurs sont correctes, c’est dans l’interprétation que tout se joue. Si on se demande :

  • Y-avait-il en moyenne plus d’émission de lumière en 1992-1993 qu’entre 2005 et 2006
    • Alors, moyenne 1992-1993 - moyenne 2005-2006.

Ensuite, la question est de savoir si cette différence est significative:

  • Dans notre cas, la p-value est de 0.33. On ne passe donc pas les seuils de significativité généralement acceptés. La différence n’est donc pas significative.

9 Analyse de la variance (ANOVA)

9.1 Pourquoi utiliser l’ANOVA ?

Lorsque l’on compare plus de deux groupes, le test t n’est plus adapté.

L’ANOVA permet de tester simultanément plusieurs moyennes. L’ANOVA nous indique si la variance entre les groupes et au sein des groupes est significativement différente.

Le principe c’est que l’ANOVA compare :

  • la variance entre groupes,
  • la variance à l’intérieur des groupes.

Quand on utilise l’ANOVA, on fait le test d’hypothèse suivant:

  • H0 : toutes les moyennes sont égales
  • H1 : au moins une moyenne diffère

Par exemple, on peut se demander si le soutien au parti pro-russe moyen est différent dans les district qui: - Ont la TV russe mais sont à plus de 25 km de la frontière. - Ont la TV russe et sont à moins de 25 km de la frontière. - N’ont pas la TV russe mais sont à plus de 25 km de la frontière. - N’ont pas la TV russe et sont à moins de 25 km de la frontière.

Dans ce cas on peut faire

Code 
# Étape 1: Créer la variable catégorielle: 

UA_precincts <- UA_precincts %>% 
  mutate(russian_tv_and_border = case_when(russian_tv==1 & within_25km==1 ~ "TV & 25",
                                           russian_tv==0 & within_25km==1 ~ "No TV & 25",
                                           russian_tv==1 & within_25km==0 ~ "TV & No 25",
                                           russian_tv==0 & within_25km==0 ~ "No TV & No 25"))

# Pour le test de l'ANOVA, c'est comme pour le t-test, on peut faire: aov(variable métrique ~ variable catégorielle, données)

# Dans notre cas on a donc: 
model <- aov(pro_russian~russian_tv_and_border, UA_precincts)

# Puis on utilise la fonction summary pour voir le résumé des résultats du modèle
summary(model)
                        Df  Sum Sq Mean Sq F value Pr(>F)    
russian_tv_and_border    3  161360   53787   169.8 <2e-16 ***
Residuals             3585 1135765     317                   
---
Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1

Ici, l’élément le plus important à analyser, c’est la p-value. On voit que la p-value est faible, on rejette donc l’hypothèse nulle et on conclut qu’il y a des différences significatives dans les moyennes entre les groupes.

En revanche, on ne sait pas pour quels groupes ces différences existent, ni dans quelle direction vont ces différences. Pour ce faire, il faut faire une comparaison de moyennes pour tous les groupes.

Cela peut être un peu fastidieux quand on a beaucoup de groupes. Heureusement, il existe une fonction qui fait cette comparaison pour vous, sur la base des résultats de votre ANOVA:

Code 
TukeyHSD(model)

10 Choisir le bon test : synthèse

Pour les analyses bivariées, vous pouvez avoir soit des variables catégorielles, soit des variables métriques. Dans chacun des cas, vous avez un test statistique (et une représentation graphique) approprié.

Le tableau suivant résume les éléments essentiels:

Variable X Variable Y Méthode Fonction R
Catégorielle. Catégorielle Chi² chisq.test()
Quantitative Quantitative Corrélation cor () & cor.test()
Catégorielle (2) Quantitative t-test t.test()
Catégorielle (3+) Quantitative ANOVA aov() & TukeyHSD()

10.1 À retenir

Pour interpréter le résultat des analyses, il faut toujours procéder avec prudence.

Avec les analyses bivariées on peut voir l’association entre deux variables et tester la significativité de cette association. Cependant:

  • un résultat significatif n’implique pas une causalité,
  • le contexte est essentiel,
  • la taille de l’effet est importante.

Si vous voulez décrire vos résultats un exemple de formulation est:

  • Il existe une association significative entre X et Y (p < 0.05).

Surtout n’oubliez pas de:

  • Toujours explorer les données avant de tester.
  • Choisir un test adapté aux variables.
  • Vérifier les conditions d’application.
  • Interpréter les résultats de manière critique.
